# 분석 거버넌스 체계 수립
## 1. 거버넌스 체계
### 1. 개요
- 거버넌스: 조직의 의사결정과 관리체계를 의미
- 분석 과정에서 발생하는 혼란을 최소화하고, 분석 목표를 효과적으로 달성하기 위해 마스터 플랜 수립 시점에서 분석 거버넌스 체계 설정이 필요
### 2. 구성 요소
- 분석 기획/관리 및 추진 조직 (Organization)
- 과제 기획 및 운영 프로세스 (Process)
- 분석 관련 인프라 및 IT 시스템 (System)
- 데이터 거버넌스 (Data Governance)
- 분석 관련 교육 및 마인드 육성 체계 (Human Resource)

## 4. 데이터 거버넌스 체계 수립
### 1. 개요
- 데이터 거버넌스: 전사 차원의 표준화된 관리 체계를 수립하고 운영을 위한 프레임워크 및 저장소를 구축하는 것
### 2. 데이터 거버넌스 구성 3요소
- 원칙: 데이터를 유지 관리하기 위한 지침과 가이드
- 조직: 데이터를 관리할 조직 및 관리자의 역할과 책임을 정의
- 프로세스: 데이터 관리를 위한 작업 절차, 모니터링 측정 등의 활동과 체계를 설정
### 3. 데이터 거버넌스 체계
1. 데이터 표준화: 용어와 이름 등을 통일하는 과정
2. 데이터 관리 체계: 메타 데이터와 데이터 사전의 관리 원칙을 수립하는 과정
3. 데이터 저장소 관리: 저장소를 구성하고 관리 대상 시스템과의 통제가 이루어지는 과정
4. 모니터링 및 개선 활동: 거버넌스 준수 여부를 확인하고, 개선활동이 이뤄지는 과정

## 5. 데이터 조직 및 인력 방안 수립
### 1. 데이터 분석 조직 구조 유형

| 구조 유형       | 특징                                                                                                                          | 
|-------------|-----------------------------------------------------------------------------------------------------------------------------|
| 집중적 조직 구조   | - 전사 분석업무를 별도의 분석전담 조직에서 담당<br>- 전략적 중요도에 따라 분석조직이 우선순위를 정해서 진행 가능하다.<br>- 이중화/ 이원화 가능성이 높다.                                |
| 기능 중심 조직 구조 | - 일반적인 분석 수행 구조<br>- 별도 분석조직이 없고 해당 업무 부서에서 분석 수행<br>- 전사적 핵심분석이 어렵다.<br>- 과거 실적에 국한된 분석 수행 가능성이 높다.                        |
| 분산 조직 구조    | - 분석조직 인력들을 현업부서로 직접 배치 하여 분석 업무 수행<br>- 전사차원의 우선순위 수행<br>- 분석결과에 따른 신속한 액션 가능<br>- 부서 분석업무와 역할분담을 명확히 해야한다. (업무과다 이원화 가능성) |
