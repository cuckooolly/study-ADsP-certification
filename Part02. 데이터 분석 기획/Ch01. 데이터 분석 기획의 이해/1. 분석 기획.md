# 분석 기획
## 1. 분석 기획의 정의와 특징
### 1. 정의
- 데이터 분석에 앞서, 주제 및 문제의 정의와 결과 도출을 위한 방안을 사전에 계획하는 작업

### 2. 분석 주제 유형 분류
1. 최적화
    - 분석 방법과 대상이 결정된 경우, 기존의 방법론은 개선하고 효율 및 성능을 높이는 데 초점을 맞춘다
2. 솔루션
    - 문제 상황과 대상이 고정되어 있으나 그 방법과 절차를 모르는 경우, 적합한 분석 방법론을 탐색한다.
3. 통찰
    - 분석 대상이 불 분명한 경우에는, 기존 분석 방법을 다양한 상황과 데이터에 적용하여 문제를 도출하는 시도를 할 수 있다.
4. 발견
    - 분석 대상을 새롭게 탐색하고 문제를 도출한 다음, 적합한 분석 방법을 선택한다.

### 4. 분석 기획 시 고려사항
1. 데이터 확보
    - 분석 주제에 적합한 데이터가 확보 가능한지를 확인
2. 활용 방안 설정 및 사례 탐색
    - 분석의 결과로 얻을 수 있는 정보와 인사이트를 비즈니스에 적용하고 활용할 수 있는 방안을 사전에 탐색하고 설정해야 한다.
    - 유사 사례를 탐색하고, 기대 효과 등을 포함한 시나리오를 설정
3. 장애 요소 파악
    - 분석 전반에 걸친 기술적인 요소와 비즈니스 요인을 종합적으로 고려하여 발생 가능한 장애 요소를 사전에 파악하고 대응 계획 수립이 필요.
