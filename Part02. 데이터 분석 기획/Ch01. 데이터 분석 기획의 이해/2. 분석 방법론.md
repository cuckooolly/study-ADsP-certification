# 분석 방법론
## 1. 분석 방법론 개요
### 1. 분석 방법론의 정의와 목적
1. 분석 방법론 활용 목적
   - 고정관념, 편향된 생각, 프레이밍 효과 등의 장애요소를 배제하고 합리적인 의사결정을 위해 데이터 분석을 활용하고, 효과적인 적용을 위해 적절한 분석 방법론을 채택
2. 분석 방법론 정의와 구성
   - 효과적인 데이터 분석을 위해 절차와 벙법을 체계적으로 정리한 것
   - 절차, 방법, 도구와 기법, 템플릿과 산출물로 구성
### 2. 대표적인 방법론 모델
1. 폭포수 모델
    - 단계별로 순차적으로 진행되는 전통적인 소프트웨어 개발 방법론
2. 나선형 모델
    - 반복적인 개발과 위험 관리를 강조하는 방법론
3. 프로토타입 모델
    - 초기 프로토타입을 개발하여 사용자 피드백을 반영하는 방법론
4. 애자일 방법론
    - 유연하고 반복적인 개발을 강조하는 방법론
    - 스프린트 단위로 작업을 나누어 빠르게 결과물을 도출하고, 지속적인 피드백과 개선을 통해 프로젝트를 진행

## 2. KDD 분석 방법론 vs CRISP-DM 분석 방법론
### 1. 분석 방법론의 등장 배경
- 운영 목적의 데이터 증가
- 데이터 활용 전략의 확대

### 2. KDD 분석 방법론
1. 데이터 선택
    - 분석에 필요한 데이터를 선택하고 목표 데이터를 구성한다.
2. 데이터 전처리
    - 잡음, 이상치, 결측치를 식별하고 제거하거나 처리한다.
    - 분석에 적합한 형태로 데이터를 정제하고 변환한다.
3. 데이터 변환
    - 분석 목적과 알고리즘에 맞게 데이터를 변환한다.
4. 데이터 마이닝
    - 적절한 분석 기법과 알고리즘을 선택하여 데이터를 분석하고 결과를 생성
5. 결과 평가
    - 분석 결과를 해석하고 분석 목적에 맞는지 확인하며 비즈니스 적용 가능성을 평가

### 3. CRISP-DM 분석 방법론
1. 비즈니스 이해
    - 프로젝트의 목적과 상황, 요구사항을 비즈니스 관점에서 이해하고 목표를 설정하고 프로젝트 계획을 수립
2. 데이터 이해
    - 데이터 수집하고 데이터 탐색을 통해 속성을 이해하고 데이터 품질 문제를 확인
3. 데이터 준비
    - 데이터 선택과 전처리 등을 통해 분석 방법에 적합한 데이터를 준비
4. 모델링
    - 다양한 모델링 기법과 알고리즘을 선택하고 모델의 성능을 테스트하고 평가하며, 파라미터를 최적화하거 과적합 등의 문제를 발견하고 대응
5. 평가
    - 완성된 모델이 프로젝트 목적에 맞는지 평가, 데이터 마이닝의 결과를 수용할지를 판단
6. 배포
    - 완성된 모델을 실제 업무나 서비스에 적용하고, 모니터링과 유지보수 계획을 수립하고 결과보고서를 작성
<br>

### 4. 빅데이터 분석 방법론
1. 분석 기획
    - 비즈니스를 이해하고 문제를 파악하여 분석 프로젝트의 범위를 설정하는 단계
2. 데이터 준비
    - 분석 기획에 근거하여 비즈니스 요구사항을 데이터 차원에서 파악하고 데이터 마트를 구성
3. 데이터 분석
    - 프로젝트 목표와 수행 계획, 확보된 데이터를 활용하여 실제 데이터 분석 프로세스를 진행
4. 시스템 구현
    - 분석 결과를 시스템에 적용하거나 프로토타입을 생성하고자 하는 경우 시스템 구현(개발) 단계를 수행할 수 있고 설계 및 구현, 테스트를 거친다.
5. 평가 및 배포
    - 데이터 분석의 결과물 등을 대상으로 기대한 성능 및 성과를 달성했는지 평가하고 적용
    - 프로젝트를 객관적/정량적으로 평가하고 중간 산출물을 정리하며, 종료 보고서를 작성하여 보고하고 프로젝트를 종료

## 3. 분석 과제 발굴
### 1. 분석 과제 설정 방식
- 하향식 접근 or 상향식 접근
- 더블 다이아몬드 프로세스와 같이 상향식과 하향식을 상호보완적으로 활용

### 2. 하향식 접근 방식

| 단계     | 설명                  |
|--------|---------------------|
| 문제 탐색  | 비즈니스 모델 내 문제 탐색     |
| 문제 정의  | 문제를 데이터 요구사항으로 변환   |
| 해결책 탐색 | 데이터 문제 해결을 위한 전략 개발 |
| 타당성 검토 | 가장 실현 가능한 솔루션 선택    |

### 3. 상향식 접근 방식
- 문제를 정의하고 주제를 설정하여 해결방안을 탐색하는 과정을 반복하며, 지속적으로 개선하는 방법

## 4. 분석 프로젝트 관리
### 1. 분석 프로젝트 관리
- 분석 프로젝트의 고려 사항
1. 데이터의 양
2. 데이터 복잡도
3. 속도
4. 분석 복잡도
5. 정확도와 정밀도
- 정확도: 표적 한가운데를 맞추는 것 / 정밀도: 영점이 얼마나 잘 맞춰져 있는가

### 2. 분석 프로젝트 관리 방안
| 관리 영역 | 분석 프로젝트 특성 및 주요 관리 항목                                                                                                                   |
|-------|-----------------------------------------------------------------------------------------------------------------------------------------|
| 범위    | 프로젝트가 진행되면서 데이터의 형태와 양, 활용 알고리즘이 자주 수정되고, 기획 단계의 수행 범위를 벗어나거나 변경됨<br>분석 최종 결과물의 형태가 다양할 수 있고, 이에 따라 투입되는 자원의 규모가 달라지므로 사전에 충분한 고려가 필요함. |
| 시간    | 데이터처리 및 준비가 지연되거나 분석 결과가 기대 수준에 못 미치는 경우 예상 보다 더 많은 시간이 소요 될 수 있음<br>사전에 설정된 프로세스와 일정을 중싱으로 Time Boxing 기법 등을 활용한 일정관리 및 시간 준수 필요       |
| 비용    | 데이터 및 인프라 비용 등을 충분히 고려<br>외부 데이터 및 상용 도구 활용에 따른 추가 비용 발생 가능성을 고려                                                                        |
| 품질    | 프로젝트 수행 결과물에 대한 품질 목표와 관리 지표를 사전에 확정해야 함<br>품질 통제와 품질 보증으로 구분하여 품질을 관리                                                                  |
| 통합    | 프로젝트와 관련된 다양한 활동과 프로세스들이 통합적으로 운영될 수 있도록 관리                                                                                             |
| 조달    | 프로젝트 목적과 상황에 알맞은 인프라 외부 소싱을 적절하게 활용                                                                                                     |
| 자원    | 프로젝트 수행 전 전문가 확보 검토 필요                                                                                                                  |
| 리스크   | 분석에 필요한 데이터 및 인프라, 프로젝트 수행 지연 가능성을 사전에 검토<br>데이터 및 알고리즘의 한계로 인해 품질 목표 달성이 어려울 수 있으며 대응 전략을 수립                                           |
| 의사소통  | 모든 프로젝트 이해관계자가 정보를 공유할 수 있는 다양한 의사 소통체계와 방법 마련 필요                                                                                       |
| 이해관계자 | 프로젝트에는 다양한 영역의 전문가가 협업을 하므로, 각 이해관계자의 식별과 관리가 필요하고, 원활한 협업을 위한 의사소통이 중요                                                                 |
